# Frontend 分析面板白话说明（给评委）

适用场景：你在 Demo 时快速解释页面，不需要技术背景。

## 1. 这个页面在做什么（一句话）

这是一个“AI 互相说服”的实验看板：看谁被说服、用了什么话术、结果有没有上链记录。

## 2. 评委先看哪 3 个位置

1. 顶部核心数字  
- 已转化人数：当前有多少 Agent 被说服  
- 回合数：已经进行了几轮辩论  
- 总投资（PBT）：被说服后投入的实验代币总量

2. 中间“策略效果分析”  
- 4 种策略：逻辑论证、情感打动、社会证明、奇迹叙事  
- 每个条显示成功率，说明不是重复话术，而是有策略实验

3. 右下角 Debate Arena（辩论区）  
- 显示“本轮谁说服谁、用了什么策略、是否成功”  
- 这就是系统的核心行为证据

## 3. 每个面板的通俗解释

- Agents：实验里一共多少个 AI 角色（固定 6 个）
- 已转化：已接受“牛市预演教”叙事的角色数量
- 总投资：转化后累计投入的 PBT（实验内指标）
- 精神市值：叙事热度和信念强度的综合分，不是现实币价
- 传教次数：被转化后继续去说服别人的次数
- 叛教次数：转化后又退出/回退的人数
- 联盟：同阵营角色结盟的次数
- 回合数：系统自动完成的辩论轮数
- 叙事主导力：当前是“怀疑叙事”还是“牛市叙事”占上风
- 策略效果分析：4 类说服策略各自成功了多少次
- Agent 网络：每个角色当前立场、阶段、PBT、任务奖励
- Debate Arena：每轮对话详情和结果（最关键）

## 4. 你可以直接照读的 30 秒讲解词

“这个页面是 AI 说服实验的实时看板。  
上面看结果：现在转化了几个人、跑了几轮、累积多少 PBT。  
中间看方法：4 种说服策略的成功率，证明系统在做策略实验，不是随机聊天。  
右边看证据：每轮辩论谁说服谁、是否成功、产生了什么变化。  
所以评判标准很直接：能不能稳定转化至少 3 个 Agent，并展示多策略有效性。”

## 5. 常见误解（评委问到时可直接回答）

- “这是投资建议吗？”  
不是。页面所有内容是 Simulation / Fiction，不构成投资建议。

- “精神市值是不是币价？”  
不是。它是实验内的叙事热度指标，用来观察行为变化。

- “你们怎么证明不是硬编码结果？”  
看 Debate Arena 每轮记录 + 策略统计 + 可选链上交易哈希（mint/event）。
